# Лабораторная работа: GUI на tkinter

В проекте сделано оконное приложение для работы со списком. Интерфейс написан на Python через `tkinter`, а сам список можно запускать в трёх вариантах:

- чистый Python;
- C++ на динамической памяти;
- C++ на STL-контейнере `std::list`.

Кнопка `Показать все` убрана. Список в окне обновляется автоматически после создания, добавления и удаления элементов.

## Папки и файлы

- `Python/app.py` — главное окно программы: кнопки, поля ввода, сообщения и вывод списка.
- `Python/list_module.py` — реализация циклического односвязного списка на Python.
- `Python/cpp_dynamic_module.py` — Python-обёртка для DLL `CPP/dynamic/list_dynamic.dll`.
- `Python/cpp_stl_module.py` — Python-обёртка для DLL `CPP/stl/list_stl.dll`.
- `CPP/dynamic/list_dynamic.cpp` — C++ реализация циклического односвязного списка через `new`, указатели и `delete`.
- `CPP/stl/list_stl.cpp` — C++ реализация списка через готовый контейнер `std::list`.

## Как запускать без `.bat`

Команды ниже нужно вводить в терминале PowerShell или cmd.

### 1. Python-вариант

1. Перейти в папку программы: `cd /d E:\proga\Python`
2. Запустить: `python app.py`

Если команда `python` не работает, можно попробовать: `py app.py`

Можно явно указать Python-вариант: `python app.py python`

### 2. C++ dynamic-вариант без `.bat`

Сначала нужна DLL `list_dynamic.dll`. Если она уже есть в `CPP/dynamic`, сборку можно пропустить.

Сборка вручную:

1. Перейти в папку C++ файла: `cd /d E:\proga\CPP\dynamic`
2. Собрать DLL: `g++ -shared -static -static-libgcc -static-libstdc++ -o list_dynamic.dll list_dynamic.cpp`

Потом запуск:

1. Перейти в папку Python: `cd /d E:\proga\Python`
2. Запустить dynamic-вариант: `python app.py dynamic`

Если `g++` не найден, значит MinGW не добавлен в `PATH`. Тогда надо либо добавить `g++` в `PATH`, либо запускать полный путь к `g++.exe`.

### 3. C++ STL-вариант без `.bat`

Сначала нужна DLL `list_stl.dll`. Если она уже есть в `CPP/stl`, сборку можно пропустить.

Сборка вручную:

1. Перейти в папку C++ файла: `cd /d E:\proga\CPP\stl`
2. Собрать DLL: `g++ -shared -static -static-libgcc -static-libstdc++ -o list_stl.dll list_stl.cpp`

Потом запуск:

1. Перейти в папку Python: `cd /d E:\proga\Python`
2. Запустить STL-вариант: `python app.py stl`

## Чем отличаются `list_dynamic.cpp` и `list_stl.cpp`

### `CPP/dynamic/list_dynamic.cpp`

Это ручная реализация циклического односвязного списка.

Главные особенности:

- есть структура `Node`;
- каждый элемент хранит `data1`, `data2` и указатель `next`;
- последний элемент указывает обратно на `head`, поэтому список циклический;
- память выделяется вручную через `new`;
- память освобождается вручную через `delete`;
- строка хранится как массив `char data2[100]`;
- для строк используются функции `strncpy` и `strcmp`.

Этот вариант лучше показывает, как список устроен внутри: указатели, голова списка, обход по кругу, удаление узлов.

### `CPP/stl/list_stl.cpp`

Это реализация через готовый контейнер `std::list`.

Главные особенности:

- вместо своих узлов используется `std::list<Item>`;
- памятью управляет сам контейнер;
- добавление в конец делается через `push_back`;
- добавление в начало делается через `push_front`;
- удаление делается через `erase`;
- строка хранится как `std::string`;
- код короче и проще.

Этот вариант меньше показывает внутреннее устройство списка, зато он безопаснее и ближе к обычному современному C++.

### Главное различие коротко

- `list_dynamic.cpp` — список написан вручную, больше работы с указателями и памятью.
- `list_stl.cpp` — список сделан через готовый контейнер STL, код проще и надёжнее.

## Как программы работают вместе

1. Пользователь запускает `Python/app.py`.
2. `app.py` создаёт окно с кнопками и полями ввода.
3. При запуске выбирается реализация списка:
   - `python` — подключается `list_module.py`;
   - `dynamic` — подключается `cpp_dynamic_module.py`;
   - `stl` — подключается `cpp_stl_module.py`.
4. Когда пользователь нажимает кнопку, например `Добавить в конец`, `app.py` вызывает функцию `add_end` у выбранного модуля.
5. Если выбран Python-вариант, функция выполняется прямо в `list_module.py`.
6. Если выбран C++ вариант, Python-обёртка через `ctypes` вызывает функцию из DLL.
7. После изменения списка `app.py` вызывает `update_view` и заново показывает элементы в `Listbox`.

Важно: у всех трёх реализаций одинаковый набор функций: `create_structure`, `delete_structure`, `add_end`, `add_beg`, `delete_element`, `count`, `get_elements`, `search_by_value`, `search_by_first_value`, `search_by_second_value`. Поэтому интерфейс может работать с ними одинаково.

## Что умеет программа

- создать список;
- удалить список;
- добавить элемент в начало;
- добавить элемент в конец;
- удалить элемент;
- посчитать элементы;
- проверить пустоту;
- искать по числу;
- искать по строке;
- искать по двум полям;
- автоматически показывать текущий список в окне.

Если выполнить неправильную операцию, программа показывает понятное сообщение и не закрывается.
